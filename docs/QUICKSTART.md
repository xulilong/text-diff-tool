# 快速开始指南

欢迎使用专业文本对比工具！这份指南将帮助你在5分钟内上手使用。

## 📦 安装

### 方式一：直接下载使用（推荐）

1. 下载项目压缩包或克隆仓库
```bash
git clone https://github.com/yourusername/text-diff-tool.git
cd text-diff-tool
```

2. 用浏览器打开 `index.html`
```bash
# Mac
open index.html

# Windows
start index.html

# Linux
xdg-open index.html
```

3. 完成！开始使用

### 方式二：本地服务器

如果你想要一个本地服务器环境：

```bash
# 使用Python（推荐）
python -m http.server 8080
# 访问 http://localhost:8080

# 或使用Node.js
npx serve
# 访问显示的地址

# 或使用PHP
php -S localhost:8080
```

### 方式三：部署到线上

可以部署到以下平台：
- GitHub Pages
- Netlify
- Vercel
- Cloudflare Pages

## 🎯 第一次使用

### 1. 加载示例数据

点击"JSON示例"或"文本示例"按钮，加载预设的示例数据。

```
点击 [📋 JSON示例] → 自动填充左右文本 → 点击 [🔍 开始对比]
```

### 2. 查看对比结果

对比完成后，你会看到：
- **差异统计**：新增、删除、修改的数量
- **变更摘要**：详细的变更列表
- **可视化对比**：左右对照的差异视图

### 3. 切换视图模式

尝试切换不同的视图模式：
- **并排对比**：左右对照，适合宽屏
- **逐行对比**：上下排列，适合窄屏

## 💡 基础使用

### JSON对比（最常用）

1. **准备数据**
```json
// 左侧（旧版本）
{
  "name": "张三",
  "age": 25
}

// 右侧（新版本）
{
  "name": "张三",
  "age": 26,
  "city": "北京"
}
```

2. **粘贴数据**
   - 复制左边的JSON到左侧编辑器
   - 复制右边的JSON到右侧编辑器

3. **开始对比**
   - 点击"开始对比"按钮
   - 或按 `Ctrl/Cmd + Enter`

4. **查看结果**
   - 看到"新增字段: 1"（city）
   - 看到"修改字段: 1"（age: 25 → 26）

### 文件对比

1. **拖拽文件**
   - 将旧版本文件拖到左侧编辑器
   - 将新版本文件拖到右侧编辑器

2. **格式自动识别**
   - 工具会根据文件扩展名自动识别格式
   - `.json` → JSON格式
   - `.sql` → SQL格式
   - `.html` → HTML格式
   - 等等...

3. **格式化（可选）**
   - 如果代码是压缩的，点击"✨ 格式化"
   - 代码会自动美化，更容易阅读

4. **开始对比**
   - 点击"开始对比"按钮

## 🎨 格式支持

### JSON格式（推荐用于API和配置）

**特点**：字段级深度对比

**适用场景**：
- API响应对比
- JSON配置文件
- 数据结构分析

**操作**：
```
1. 粘贴JSON → 自动识别为JSON
2. 点击"格式化"（如果需要）
3. 开始对比 → 看到字段级差异
```

### SQL格式（数据库脚本）

**特点**：关键字高亮、自动换行

**适用场景**：
- 数据库DDL对比
- SQL查询对比
- 迁移脚本对比

**操作**：
```
1. 加载.sql文件 → 自动识别为SQL
2. 点击"格式化" → 关键字大写、自动换行
3. 开始对比 → 看到SQL差异
```

### HTML/XML格式（标签文档）

**特点**：标签对齐、层级清晰

**适用场景**：
- HTML模板对比
- XML配置对比
- SVG文件对比

**操作**：
```
1. 加载.html或.xml文件 → 自动识别
2. 点击"格式化" → 标签自动对齐
3. 开始对比 → 看到标签差异
```

## ⚙️ 高级功能

### 1. 忽略空白字符

```
用途：只关注实质内容变化，忽略格式差异

操作：勾选"忽略空白"选项 → 开始对比

适用：对比不同编辑器或IDE格式化的代码
```

### 2. 自动格式化JSON

```
用途：自动美化JSON，统一格式

操作：保持"自动格式化JSON"勾选（默认开启）

效果：压缩的JSON会自动格式化为2空格缩进
```

### 3. 导出对比结果

```
用途：保存对比结果，方便分享或归档

操作：点击"📥 导出"按钮 → 保存为HTML文件

用途：发送给同事、存档记录、打印
```

### 4. 交换左右文本

```
用途：快速对调左右内容

操作：点击"🔄 交换左右"按钮

效果：左右文本和格式选择都会交换
```

## 🔧 常见问题

### Q1: 如何对比压缩的JSON？
```
A: 直接粘贴 → 点击"格式化"按钮 → JSON会自动美化 → 开始对比
```

### Q2: 支持哪些文件格式？
```
A: 支持8种格式：
   - JSON, XML, HTML, SQL, CSS, JavaScript, YAML, 纯文本
   - 可以自动识别，也可以手动选择
```

### Q3: 数据会上传到服务器吗？
```
A: 不会！所有操作都在你的浏览器本地完成
   - 100%本地运行
   - 不会保存任何数据
   - 关闭浏览器后自动清除
```

### Q4: 如何对比大文件？
```
A: 工具支持最大5MB的文件
   - 如果文件太大，建议分段对比
   - 或者使用专业的diff工具（如Beyond Compare）
```

### Q5: 为什么格式识别不准确？
```
A: 可以手动选择格式
   - 使用编辑器顶部的"格式"下拉框
   - 选择正确的格式类型
```

```javascript
// 开发环境API响应
fetch('/api/dev/user').then(r => r.json()).then(data => {
  // 复制到左侧编辑器
  console.log(JSON.stringify(data, null, 2));
});

// 生产环境API响应
fetch('/api/prod/user').then(r => r.json()).then(data => {
  // 复制到右侧编辑器
```

