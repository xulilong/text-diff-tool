JSON 格式化常见问题与使用指南

JSON 常用于接口返回、配置文件和日志数据。格式化工具可以帮助你检查语法、阅读层级、压缩体积，以及处理转义字符串。

JSON 格式化失败的常见原因

  • 属性名没有使用双引号，例如 {name: "test"} 不是标准 JSON。
  • 字符串使用了单引号，标准 JSON 字符串需要双引号。
  • 数组或对象最后多了逗号，例如 {"a":1,}。
  • 包含注释，标准 JSON 不支持 // 或 /* */ 注释。
  • 接口返回内容前后混入了日志、HTML 或其他非 JSON 文本。

格式化、压缩、转义分别有什么用

  • 格式化：把一行 JSON 展开成有缩进的多行结构，方便阅读和排查字段。
  • 压缩：去掉空格和换行，把 JSON 变成更短的一行，适合复制到配置或接口参数里。
  • 转义：把普通文本转换成适合放进 JSON 字符串的形式。
  • 反转义：把带有 \"、\n 等转义字符的内容恢复为可读文本。

什么时候用树形结构查看

当 JSON 层级较深、字段较多时，树形结构比纯文本更容易定位字段。你可以展开对象、搜索字段名、复制完整 JSON 或复制某个节点内容。

处理敏感数据的建议

JSON 中经常包含 token、密钥、用户手机号或业务配置。使用公共设备时建议先脱敏，并在用完后清理浏览器本地历史。
